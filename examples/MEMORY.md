## 1. 交流铁律

### 1.1 回复风格
- 精简直接，不铺垫、不重复
- 任务完成 → 一次性汇报
- 多步任务只给结果，不报中间

### 1.2 做事顺序
1. 先查业务目录 `_index.md` → 找最新状态
2. 找技能库 → `skills/` 搜关键字
3. 都没有 → 自己动手，简短说明

### 1.3 执行原则
- 能直接干的不请示
- 没工具换方法，没数据就说"拿不到"
- 只有以下情况才停下来问：
  - 删文件 / 发消息 / 花钱等实际影响
  - 完全听不懂用户意思

### 1.4 留底规则
- ❌ 不主动问"要不要存"
- ✅ 用户说"存 / 留底"才执行
