# AI Memory Hygiene · 让 AI 记忆清爽不臃肿

> 一套面向 AI 助手重度用户（OpenClaw / Claude Code / Cursor / Cherry Studio 等）的
> 知识管理方法论，解决「AI 记忆越长越乱、找不到最新状态、不同任务数据混杂」的痛点。

## 🎯 这是什么

用 AI 助手久了，你一定遇到过这些问题：

- **记忆越来越长**：MEMORY.md 从 1KB 涨到 50KB，AI 启动变慢、回答越来越慢
- **找不到「哪个是最新的」**：旧版本和新版本堆在一起，AI 经常拿过时数据回答
- **不同任务的数据混在一起**：股票分析、论文写作、工作交接全塞一个文件
- **不敢清理**：清掉又怕丢关键规则，AI 失去上下文后行为失常
- **新会话失忆**：昨天聊的项目，今天 AI 完全不记得

**这套方法解决的就是这些问题。**

核心思想一句话：**MEMORY.md = 目录 + 铁律，详细内容各回各家**。

---

## 🧠 核心理念

```
MEMORY.md     → 身份 + 长期规则 + 目录索引（≤100 行，常驻上下文）
业务目录/     → 详细内容（每个目录配 _index.md 做导航）
memory/       → 当日要点 + 决策指针（≤30 行/文件）
tmp/          → 一次性产物（可随时清）
```

**关键原则：**

| 原则 | 解释 |
|---|---|
| **分层存放** | 规则、详细内容、当日指针、物理临时文件各归各位 |
| **索引而非堆叠** | 每个目录配 `_index.md`，标日期 + 指针，AI 自己去找最新文件 |
| **指针而非副本** | memory/ 只写"做了什么 + 指向哪"，不复制业务目录的详细内容 |
| **定期清理** | tmp/ 随时清、work_records/ 每月清、MEMORY.md 季度体检 |
| **自定义指标必须说明** | 报告中自己造的评级/打分要附定义，避免被读不懂 |

## 📁 推荐目录结构

```
workspace/
├── MEMORY.md                    # ≤100 行：身份 + 铁律 + 目录索引
│
├── 股票/                        # 业务目录：按主题分
│   ├── _index.md                # 导航：标的 → 最新状态文件
│   ├── 持仓跟踪/
│   ├── 操作记录/
│   ├── 规则档/
│   └── 复盘/
│
├── research/                    # 另一个业务目录
│   ├── _index.md
│   ├── 论文1/
│   └── 数据/
│
├── work_records/                # 跨会话任务交接
│   ├── _index.md
│   └── YYYY-MM-DD_任务名.md
│
├── memory/                      # 当日要点 + 决策指针
│   ├── _index.md                # 总索引
│   └── YYYY-MM-DD.md            # 当日记录，≤30 行
│
└── tmp/                         # 临时文件
    └── (csv / xlsx / png / 临时脚本)
```

**关键约束：**

- `MEMORY.md` ≤ 100 行，常驻上下文，每次 AI 启动都要读
- 业务目录的 `_index.md` ≤ 50 行，按日期倒序列出文件
- `memory/` 每个文件 ≤ 30 行，超过就拆出去

## 📜 三大铁律

### 铁律 1：身份与规则放 MEMORY.md

只放三类内容：

```
## 0. 身份（机器人 + 服务对象 + 时区）
## 1. 交流铁律（回复风格 + 做事顺序 + 执行原则）
## 2. 业务目录索引（每个业务目录 + 一句话说明）
## 3. 写作风格（如果长期不变）
## 4. 已安装技能列表（如需要）
## 5. 清理节奏（提醒定期清理）
```

**绝对不放：**

- ❌ 聊天记录、对话摘要
- ❌ 当前持仓、最新分析（变动数据）
- ❌ 详细的 API 密钥（放 TOOLS.md，需要时才读）
- ❌ 超过 100 行的内容

### 铁律 2：详细内容按业务分类

按你**实际的工作主题**分目录，而不是按文件类型。

**错误示范**：

```
❌ /md_files/
❌ /reports/
❌ /data/
```

**正确示范**：

```
✅ /股票/        # 所有跟股票相关的都在这里
✅ /research/    # 所有跟论文相关的都在这里
✅ /work_records/  # 所有跨会话任务交接都在这里
```

每个业务目录配 `_index.md`，AI 找数据时先扫索引：

```markdown
# 股票目录索引

## 当前状态指针
- 最新持仓：`持仓跟踪/今日状态（2026-07-02）_v1.md`
- 最新复盘：`复盘/复盘报告_2026-07-02.md`
- 当前规则：`规则档/V4.4.5_交易规则_20260702.md`

## 历史
- V4.4.4 (2026-07-01)
- V4.4.3 (2026-07-01)
```

### 铁律 3：memory/ 只放指针

memory/ 是**当天发生了什么**的极简记录，不写详细内容。

**示例 `memory/2026-07-05.md`：**

```markdown
# 2026-07-05

## 今天做了什么
- 整理工作区目录（详见 work_records/2026-07-05_目录重构.md）
- 股票分析三主题（详见 work_records/2026-07-05_股票分析专题.md）

## 关键决策
- 启用新目录结构
- 删 MEMORY_旧版备份.md

## 当前生效
- 规则：V4.4.5
- 持仓快照：7-02
```

**不写：**

- ❌ 详细数据表格
- ❌ 完整分析内容
- ❌ 超过 30 行

## 💡 实战示例

### 场景 1：股票投资者

```
股票/
├── _index.md              ← 导航
├── 持仓跟踪/
│   ├── _模板.md           ← 每日快照模板
│   └── 今日状态（2026-07-02）_v1.md
├── 操作记录/
│   └── 操作日志.md
├── 规则档/
│   ├── V4.4.5_交易规则_20260702.md  ← 当前
│   └── V4.4.4_交易规则_20260701.md  ← 历史
├── 复盘/
│   └── 复盘报告_2026-07-02.md
└── 回测报告/
```

**AI 工作流：**

1. 用户：「今天怎么操作？」→ AI 打开 `规则档/V最新.md` + `持仓跟踪/今日状态`
2. 用户：「复盘」→ AI 看 `复盘/复盘报告_最新.md`
3. 用户：「清 tmp」→ AI 删 `../tmp/` 下所有文件

```
├── project_a/
│   ├── _index.md

├── _index.md
```

### 场景 4：跨会话任务交接

```

├── _index.md
├── 2026-07-05_目录重构.md
```

新会话开始时，AI 扫一遍 `_index.md` 就能立刻知道：
- 上次任务是什么
- 当前进行到哪一步
- 有没有遗留问题

## 📊 自定义指标的说明义务

**这是新手最容易踩的坑。**

你在报告中用了自己造的评级，比如：

> 「海光信息：弹性=极高」

读者（包括 AI 助手自己过几天回看）会问：
- 「弹性」是什么？
- 怎么算出来的？
- 极高是多高？

**正确做法：附简要定义（三要素）**

```markdown
### 弹性指标说明
- **含义**：股价潜在涨幅空间，由业务弹性、估值弹性、题材弹性综合评估
- **计算依据**：
  - 业务弹性：主业增速、订单/产能扩张速度
  - 估值弹性：当前估值 vs 业绩预期
  - 题材弹性：政策/技术/事件催化想象空间
- **取值范围**：极高 / 高 / 中等 / 偏低 / 低（五档）
```

**其它常见自定义指标：**

- 风险等级
- 估值水位
- 情绪热度
- 信号强度
- 业绩确定性

每个第一次出现时都加一行说明。

## 🧹 清理节奏

| 目录 | 清理频率 | 触发条件 |
|---|---|---|
| `tmp/` | 随时 | 用户说「清 tmp」 |
| `work_records/` | 每月 | 30 天前的过期记录 |
| `memory/` | 每周 | 合并/清理过期条目 |
| `股票/复盘/` | 月底 | 归档到 `复盘归档/YYYY-MM/` |
| `MEMORY.md` | 季度 | 体检：删除已失效规则 |
| 业务目录内 | 按需 | 新版本出来后旧版可归档 |

**清理原则：**

- 删除前确认「业务目录里有完整版」
- 保留至少一份历史快照（如月度归档）
- MEMORY.md 体检时删「已不再适用」的规则

## 🚀 迁移指南（从「一团乱」到「清爽」）

如果你已经在用 AI 但记忆管理混乱，按这四步走：

### Step 1：备份旧的 MEMORY.md

```bash
cp MEMORY.md MEMORY_旧版备份.md
```

不要急着删，先备份到 `backups/` 目录。

### Step 2：提取长期规则

问自己这些问题：

- 「这个规则一周后还会生效吗？」
  - 是 → 留 MEMORY.md
  - 否 → 搬到业务目录
- 「这是身份信息还是变动数据？」
  - 身份（我叫什么、用户是谁、时区）→ MEMORY.md
  - 变动（当前持仓、最新分析）→ 业务目录

**重构后的 MEMORY.md 模板：**

```markdown
# MEMORY.md

## 0. 身份
- 机器人：XXX
- 用户：YYY
- 时区：Asia/Shanghai

## 1. 交流铁律
- 精简直接
- 任务完成一次性汇报
- 多步任务只给结果

## 2. 业务目录索引
- 股票 → `股票/`
- 论文 → `research/`
- 工作交接 → `work_records/`

## 3. 写作风格
- 排比递进、四字结构

## 4. 已安装技能
- `skills/` 目录

## 5. 清理节奏
- tmp/ 随时清
- work_records/ 每月清
```

### Step 3：按主题建业务目录

```
mkdir -p 股票 research work_records memory tmp
```

把原来的散落文件按主题分类移动到对应目录。

### Step 4：建 _index.md

每个业务目录配一个：

```markdown
# 业务目录索引

> 最后更新：YYYY-MM-DD

## 当前状态指针
- 最新 XXX：`路径/文件名.md`
- 当前规则：`路径/文件名.md`

## 历史
- v1 (YYYY-MM-DD)
- v2 (YYYY-MM-DD)

## 操作流程
1. 用户问 X → 打开哪个文件
2. 用户问 Y → 打开哪个文件
```

## 🔧 工具适配

这套方法适用于任何支持文件系统的 AI 助手：

### OpenClaw（最契合）

- 内置 `MEMORY.md` + `memory/` 加载机制
- `skills/` 目录自然契合业务目录
- `cron` + `heartbeat` 提醒清理

### Claude Code / Cursor / Windsurf

- 项目根目录的 `CLAUDE.md` / `.cursorrules` 对应 MEMORY.md
- 子目录结构完全适用
- 用 `@filename` 引用文件时 AI 会自动读取

### Cherry Studio / ChatGPT

- 每次对话粘贴 MEMORY.md 内容
- 用「对话模板」功能预设索引结构
- 长任务拆成多段，每段开头粘索引

### 其他 AI（通用原则）

- 把 MEMORY.md 内容放在系统提示或开头
- 业务目录文件用工具调用读取
- 不要把全部内容塞进一个 prompt

## 🎓 设计哲学

### 为什么不把全部内容都放 MEMORY.md？

1. **Token 成本**：MEMORY.md 每次启动都加载，50KB 就是每次都付钱
2. **注意力分散**：AI 会在海量内容里"迷失"，反而抓不住重点
3. **更新困难**：改一处牵动全身，不如分层修改局部

### 为什么需要 _index.md？

AI 不是神，没法扫描整个目录判断哪个最新。

给它**显式的指针**（"最新在 XXX"），它就能：
- 1 秒找到
- 不会读错版本
- 不会瞎猜

### 为什么 memory/ 不直接写详细内容？

- **流动性**：今天的"重要"，明天就过时
- **检索性**：指针比副本好找
- **清理性**：删起来不心疼
